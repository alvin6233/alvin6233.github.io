监控搭建

2019-03-12 监控 阅读量

监控对于系统，尤其是分布式系统的重要性不言而喻。这里不多做概念的阐述，主要讲目前公司使用的监控架构简单的自动动手实现下。本文主要分两个重要模块，
一个是部署美团点评开源的分布式监控系统cat，另一个作为业务打点补充statsd+graphite+grafana构建业务打点监控系统。目前的这种结构是有些冗余的，后续会慢慢探索优化。

cat部署（集群模式）

实战

cat版本：v3.0.0

系统：172.16.6.177（centos7）、172.16.6.178（centos7）

服务器：tomcat8

下载源代码，构建运行包

若下载源码比较慢，可直接下载war包。

tomcat配置（每个服务器）

  • 调整启动参数，修改 catalina.sh文件

    i. 需要每台CAT集群172.16.6.177、172.16.6.178都进行部署
    ii. 建议使用cms gc策略
    iii. 建议cat的使用堆大小至少10G以上，开发环境启动2G堆启动即可（自己玩就随便喽）

  • 修改中文乱码 tomcat conf 目录下 server.xml

相关目录及配置文件

  • 创建目录（每个系统）

  • 配置/data/appdatas/cat/client.xml（每台）

    i. 此配置文件的作用是所有的客户端都需要一个地址指向CAT的服务端
    ii. 此文件可以通过运维统一进行部署和维护，比如使用puppet等运维工具
    iii. 不同环境这份文件不一样，比如区分prod环境以及test环境，在美团点评内部一共是2套环境的CAT，一份是生产环境，一份是测试环境
    iv. 注意：当路由错误，且该文件夹下存在client_cache.xml，请删除client_cache.xml，再重启服务

  • 安装CAT的数据库

数据库的脚本文件 script/CatApplication.sql

MySQL的一个系统参数：max_allowed_packet，其默认值为1048576(1M)，修改为1000M，修改完需要重启mysql

注意：
i. 集群共用一个数据库
ii. 数据库编码使用utf8mb4，否则可能造成中文乱码等问题

  • 配置/data/appdatas/cat/datasources.xml（每台）

war部署

  • 将cat.war部署到tomcat下的webapps，启动tomcat

  • 打开控制台的URL，http://172.16.6.177:8080/cat/s/config?op=routerConfigUpdate

  • 默认用户名：admin 默认密码：admin

问题总结

事实上并非这么顺利，中间遇到了很多蛋疼的问题，泪崩。

177服务器，外部访问8080不通

statsd + graphite + grafana

由于涉及到组件较多，这里采用docker部署方式。真是方便！！！！

系统：172.16.6.91（centos7）

下载镜像，创建容器

事情并非想象中的简单，报错了：docker: Error response from daemon: OCI runtime create failed: container_linux.go:344: starting container process caused “process_linux.go:293: copying bootstrap data to pipe caused \”write init-p: broken pipe\””: unknown.

google了一下大体原因是centos的kernel版本太低。so只能升级kernel核心版本喽！！

具体步骤不搬了，都在这里，我是按照这个执行的，最后成功啦：https://www.howtoforge.com/tutorial/how-to-upgrade-kernel-in-centos-7-server/

容器包含的组件

Includes the following components：

  • Nginx - reverse proxies the graphite dashboard
  • Graphite - front-end dashboard
  • Carbon - back-end
  • Statsd - UDP based back-end proxy
  • Grafana - front-end dashboard (more refined than Graphite)
